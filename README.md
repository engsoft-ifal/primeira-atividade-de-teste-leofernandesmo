# Template PWEB1 IFAL

Este repositório é um **template** para atividades da disciplina Programação Web 1 (PWEB1) do IFAL. 
Foi concebido para servir como base padronizada para exercícios, trabalhos e entregas via GitHub Classroom. 
O objetivo é fornecer uma estrutura mínima, instruções claras de execução local e orientações para submissão, facilitando a avaliação e a reprodução do trabalho pelos docentes.

---

## Estrutura do repositório

A estrutura proposta é intencionalmente simples e adequada a alunos iniciantes:

```
pweb1-cap01-primeirorepo/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── script.js
├── assets/
│   ├── images/
│   └── fonts/
├── README.md
└── .gitignore
```

**Descrição dos diretórios e arquivos**

- **index.html** — ponto de entrada da aplicação; deve conter a estrutura semântica básica.  
- **css/style.css** — arquivo principal de estilos.  
- **js/script.js** — arquivo principal de scripts.  
- **assets/** — recursos estáticos como imagens e fontes.  
- **README.md** — documentação do projeto (este arquivo).  
- **.gitignore** — lista de arquivos e pastas que não devem ser versionados.

---

## Requisitos mínimos

- Editor de código (recomendado Visual Studio Code).  
- Navegador moderno (Chrome, Firefox, Edge ou Safari).  
- Git instalado para controle de versão.  

---

## Como executar localmente

### Abrir sem servidor
1. Abra a pasta do projeto no seu gerenciador de arquivos.  
2. Clique duas vezes em `index.html` para abrir no navegador.  

---

## Instruções de submissão via GitHub Classroom

1. Aceite o assignment no GitHub Classroom para gerar seu repositório privado.  
2. Clone o repositório gerado para sua máquina local.  
3. Implemente a atividade na estrutura mínima fornecida.  
4. Faça commits regulares documentando seu progresso.  
5. Ao finalizar, faça `push` para o repositório remoto.  
6. Verifique se todos os arquivos necessários estão presentes e se o `README.md` contém instruções de execução.

---

## Critérios de avaliação (resumo)

A avaliação das atividades práticas seguirá critérios objetivos. Abaixo está um resumo que orienta a entrega:

- **Estrutura** — presença dos arquivos mínimos (`index.html`, `css/style.css`, `js/script.js`, `assets/`).  
- **Funcionalidade** — a página abre e os requisitos da atividade funcionam conforme especificado.  
- **Qualidade do código** — HTML semântico, CSS organizado, JavaScript legível e comentado quando necessário.  
- **Documentação** — `README.md` com instruções claras de execução.  
- **Boas práticas** — nomes de arquivos coerentes, imagens otimizadas, ausência de segredos versionados.

Detalhes e pesos serão informados no enunciado de cada atividade.
